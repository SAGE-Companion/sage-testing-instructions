What's New in This Build

Elder and Caretaker workflows are now in testing ahead of launch next week. Key updates since the last round:

💬 Support chat — message auto-creates a ticket, admin replies and closes from /admin/support, closed tickets archived, new message opens a fresh ticket.
✅ Elder consent via email — Elder receives a confirmation email when Caretaker adds them and must click to consent before access is granted.
📋 Consolidated booking view for Caretakers — all bookings across linked Sages visible directly without impersonation.
📅 Multi-date & recurring bookings now supported.
🔤 Confusing terminology updated — "Active Sages," "View as Sage," "Primary caretaker," care role labels revised for clarity.
🔄 Companion onboarding flow fully revamped — sign-up, last screen, and admin bar UX improved.
🏦 Companion can accept up to 3 bookings before Stripe identity verification is complete.
⏰ Check-in flow updated — "On My Way" available anytime, check-in button unlocks 5 minutes before booking start.
💰 Pricing updated — late-night (10pm–6am) and weekend (Sat/Sun) premium pricing applies a flat add-on amount on top of each companion's base rate per service. The add-on amount is set by admin from the Pricing tab and applies uniformly across all services (e.g. a companion with a $35 base rate becomes $45 during premium hours if the add-on is $10).
🇪🇸 Spanish language support — sign-up, login, cancellation policy, terms, and refund policy pages.
🚫 Application rejection — screens now show a general message only; reason/notes stripped from emails and SMS but retained in admin panel.
📧 Caretaker email change — verified via link to old inbox, all sessions revoked on change, user signs back in with new email.
⚖️ Dispute resolution — Elder files dispute or auto-created on denial, payment held, admin reviews and picks resolution (full refund / full pay / 50–50 split), Stripe executes automatically.
🚫 No-show policy — account auto-deactivated; admin can manually reactivate.
🛡️ Emergency cancellation exceptions — admin can apply override in admin panel to waive the 50% charge.

⚠️ Browser Setup — Avoid Login Conflicts

You'll be testing multiple user types simultaneously. Each needs its own isolated browser session.

  • Profile 4 — Caretaker
Different Browsers
  • Chrome — Admin
  • Safari — Caregiver
  • Firefox — Elder
  • Edge — Caretaker
Incognito Windows
  • Regular — Admin
  • Incognito 1 — Caregiver
  • Incognito 2 — Elder

Testing Checklist

This checklist covers only new and updated features since the last testing round. Previously verified items are not included. Focus your testing here.

Elder

  • Elder consent email — add as a Caretaker's elder and verify confirmation email arrives and consent button works
  • Multi-date / recurring booking — create a booking across multiple dates and verify it saves correctly
  • Late-night premium (10pm–6am) — verify premium pricing applied at checkout
  • Weekend premium (Sat/Sun) — verify premium pricing applied at checkout
  • Support chat — send a message, verify ticket auto-created, receive admin reply
  • After admin closes ticket — verify chat clears and welcome message appears
  • Send new message after closure — verify a new ticket opens
  • Dispute — file a dispute after checkout, verify payment is held and booking moves to Disputed state
  • Dispute resolved — verify no rating or strike applied
  • Spanish — verify sign-up, login, cancellation policy, terms, and refund policy display in Spanish
  • Application rejection screen — verify only general message shown, no reason or notes visible

Caretaker

  • Add elder — verify elder receives consent confirmation email
  • Elder clicks email consent — verify Caretaker access is granted only after confirmation
  • Consolidated booking view — verify all bookings across linked Sages visible without impersonation
  • Booking marked as caretaker-initiated — verify companion can see it was booked on behalf of an elder
  • Email change — enter new email, verify link sent to old inbox, click link, verify all sessions revoked and sign-in with new email works
  • Support chat — open ticket, receive reply, verify 1 active ticket limit
  • Spanish — verify key pages display correctly in Spanish

Caregiver (Companion) — Light Recheck

  • "On My Way" button available anytime on upcoming booking
  • Check-in button unlocks 5 minutes before booking start
  • Can accept up to 3 bookings before Stripe verification complete
  • Support chat accessible and working
  • No strike or rating impact when dispute is raised against a booking

Admin

  • Support tickets visible in /admin/support — reply to user, close ticket
  • Closed tickets visible in Closed tab for history
  • Disputed bookings visible in /admin/bookings Disputed tab
  • Review dispute — check-in/out logs, photos, caregiver notes
  • Resolve dispute — full refund to elder
  • Resolve dispute — full pay to caregiver
  • Resolve dispute — 50/50 split — verify Stripe executes automatically and ledger updates
  • Rejected application — verify reason and notes visible in admin but not shown to applicant
  • Emergency cancellation override — apply waiver for 50% charge
  • No-show — manually reactivate a deactivated account
